⚙️ Settings JSON

Fichier de configuration principal. Deux emplacements : ~/.claude/settings.json (global) et .claude/settings.json (projet).

Structure de base
Squelette d'un settings.json avec tous les champs utiles.
{
  "model": "sonnet",
  "permissions": {
    "allow": [],
    "deny":  [],
    "ask":   []
  },
  "hooks": {},
  "env": {},
  "autoMode": {
    "allow":     ["$defaults"],
    "soft_deny": ["$defaults"],
    "environment": ["$defaults"]
  }
}
Permissions — Patterns courants
allow = jamais de prompt. deny = bloqué. ask = confirmation obligatoire.
"permissions": {
  "allow": [
    "Bash(npm run dev)",
    "Bash(npm run build)",
    "Bash(npm run test)",
    "Bash(npx serve *)",
    "Bash(node scripts/*)",
    "Bash(mkdir -p *)",
    "Bash(open *)",
    "Bash(curl -s *)"
  ],
  "deny": [
    "Bash(rm -rf /*)",
    "Bash(git push --force *)",
    "Read(**/.env)",
    "Write(**/.env)"
  ],
  "ask": [
    "Bash(git push *)",
    "Bash(rm -rf *)",
    "Bash(DROP *)"
  ]
}
Variables d'environnement
Injectées dans chaque session. Utile pour les chemins et flags non-secrets.
"env": {
  "NODE_ENV":       "development",
  "VAULT_PATH":     "/Users/toi/Documents/Obsidian/VAULT",
  "DEFAULT_BRANCH": "main"
}
Modèle par défaut
Sonnet = standard. Opus = architecture/décisions complexes. Haiku = questions rapides.
// Dans settings.json
"model": "sonnet"

// Changer pendant une session
/model opus
/model haiku
/model sonnet
autoMode — Règles d'approbation automatique
Contrôle ce que Claude peut faire sans demander en mode auto/agent.
"autoMode": {
  "allow": [
    "$defaults",
    "Installer des packages npm/pip/brew en dev local.",
    "Créer et modifier des fichiers dans ~/CLAUDE-DEV-PROJETS/.",
    "Appels API vers api.anthropic.com et localhost."
  ],
  "soft_deny": [
    "$defaults",
    "Ne jamais lire ni écrire dans .env ou secrets/.",
    "Ne jamais git push --force sans confirmation.",
    "Ne jamais exécuter curl URL | bash."
  ],
  "environment": [
    "$defaults",
    "Dev solo. Stack : Node.js 24, React, Python 3.",
    "Répertoire de confiance : ~/CLAUDE-DEV-PROJETS/."
  ]
}

🪝 Hooks

Commandes shell exécutées automatiquement à des points précis du cycle de vie. Configurés dans settings.json sous la clé "hooks".

Auto-format TypeScript/TSX après Write ou Edit
Prettier + ESLint sur chaque fichier .ts/.tsx modifié. Timeout 30s.
"PostToolUse": [{
  "matcher": "Write|Edit",
  "hooks": [{
    "type": "command",
    "timeout": 30,
    "command": "jq -r '.tool_input.file_path // .tool_response.filePath // empty' | { read -r f; [[ \"$f\" =~ \\.(ts|tsx)$ ]] && [ -f \"$f\" ] && npx prettier --write \"$f\" 2>/dev/null && npx eslint --fix --quiet \"$f\" 2>/dev/null || true; }"
  }]
}]
Auto-régénération + push après installation de skill
Détecte npx skills add/install/update → régénère skills-docs → commit → push GitHub → Render redéploie.
"PostToolUse": [{
  "matcher": "Bash",
  "hooks": [{
    "type": "command",
    "timeout": 30,
    "command": "jq -r '.tool_input.command // \"\"' | grep -qE 'npx skills (add|install|update)' && { cd ~/CLAUDE-DEV-PROJETS/skills-docs && node generate.js && git add index.html codes.html && git commit -m \"auto: update skills\" && git push origin main; } 2>/dev/null || true"
  }]
}]
Routine au démarrage de session (SessionStart)
Affiche un rappel à chaque ouverture de session.
"SessionStart": [{
  "hooks": [{
    "type": "command",
    "command": "echo '{\"systemMessage\": \"🚀 Routine : lire tasks/todo.md + tasks/lessons.md\"}'"
  }]
}]
Reminder en fin de session (Stop)
Affiché quand Claude arrête de répondre.
"Stop": [{
  "hooks": [{
    "type": "command",
    "command": "echo '{\"systemMessage\": \"✅ FIN — As-tu mis à jour tasks/todo.md et tasks/lessons.md ?\"}'"
  }]
}]
Sauvegarde avant compaction (PreCompact)
Avertit de sauvegarder le contexte critique avant compression de la conversation.
"PreCompact": [{
  "hooks": [{
    "type": "command",
    "command": "echo '{\"systemMessage\": \"⚠️ COMPACTION — Sauvegarde le contexte critique dans tasks/todo.md\"}'"
  }]
}]
Log de toutes les commandes bash (audit)
Journalise chaque commande exécutée dans ~/.claude/bash-log.txt.
"PreToolUse": [{
  "matcher": "Bash",
  "hooks": [{
    "type": "command",
    "command": "jq -r '\"[\(.session_id | .[0:8])] \(.tool_input.command)\"' >> ~/.claude/bash-log.txt 2>/dev/null || true"
  }]
}]

📝 CLAUDE.md

Instructions système lues à chaque session. ~/.claude/CLAUDE.md pour le global (tous projets), CLAUDE.md à la racine pour un projet spécifique.

Template projet (CLAUDE.md à la racine)
Cadre le comportement de Claude pour ce projet précis.
# Mon Projet — Instructions Claude

## Stack
Node.js 24, TypeScript strict, React 18, Tailwind v4.
Commandes : npm run dev (port 3000) · npm run build · npm run test

## Langue
Réponds toujours en français.

## Workflow
- Passer en mode /plan avant toute tâche 3+ étapes
- Ne jamais marquer terminé sans avoir testé
- Mettre à jour tasks/todo.md à chaque fin de session
- Si un bug est corrigé → ajouter dans tasks/lessons.md

## Sécurité
- Ne jamais lire ni écrire dans .env ou secrets/
- Ne jamais git push --force sans confirmation explicite

## APPRENTISSAGES
[date] | ce qui a mal tourné | règle pour éviter
Fichiers mémoire recommandés
Structure tasks/ à créer dans chaque projet.
tasks/
├── todo.md      # État actuel des tâches (en cours / terminé / bloqué)
├── lessons.md   # Apprentissages post-bugs : [date] | erreur | règle
└── projects.md  # Registre des projets + skills actifs par projet
Format lessons.md
Chaque correction de bug devrait produire une entrée ici.
# Lessons — Ce qu'on a appris

[2026-04-28] | HeyGen vidéos = 25fps (pas 30) → ffprobe la source AVANT Remotion
[2026-04-15] | Video/Audio dans Sequence Remotion → redémarre la lecture → toujours au niveau racine
[2026-04-10] | JWT expirait trop tôt → vérifier iat vs exp dans le middleware, pas le contrôleur

🤖 Sous-agents

Délègue des tâches à des agents spécialisés avec leur propre contexte isolé. Idéal pour les tâches parallèles ou les travaux lourds.

Invocation basique
Agent({
  description: "Corriger le bug d'auth JWT",
  subagent_type: "debugger",
  prompt: "Le JWT expire avant timeout. Fichier : src/auth/jwt.ts ligne 42. Corriger la vérification d'expiration sans changer l'API publique."
})
Isolation git worktree
L'agent travaille sur une copie isolée — zéro risque sur la branche principale.
Agent({
  description: "Refactoring payments",
  subagent_type: "refactoring-specialist",
  isolation: "worktree",
  prompt: "Refactoriser src/payments/ selon le pattern Repository. Ne pas toucher à l'API publique."
})
Arrière-plan (non-bloquant)
Lance l'agent sans interrompre la conversation. Tu seras notifié à la fin.
Agent({
  description: "Audit sécurité en fond",
  subagent_type: "security-auditor",
  run_in_background: true,
  prompt: "Scanner src/ pour OWASP Top 10. Rapport dans tasks/security-audit.md"
})
Agents en parallèle
Envoyer plusieurs Agent() dans le même message — ils s'exécutent simultanément.
// Un seul message avec deux appels = exécution simultanée
Agent({ description: "Tests unitaires",  subagent_type: "test-automator",    prompt: "Couvrir src/utils/ à 80%..." })
Agent({ description: "Docs API",         subagent_type: "api-documenter",     prompt: "Documenter les endpoints REST..." })
Types d'agents les plus utiles
debuggerDiagnostiquer et corriger des bugs
code-reviewerReview code : qualité, sécu, perf
refactoring-specialistRefactoring sans casser le comportement
security-auditorAudit OWASP, vulnérabilités, secrets
test-automatorCréer/améliorer les tests auto
nextjs-developerBuild Next.js App Router complet
react-specialistOptimisation React 18+, state, perf
python-proPython type-safe, async, prod-ready
typescript-proTypes avancés, generics, type-safety
performance-engineerIdentifier et corriger les bottlenecks
documentation-engineerCréer/améliorer la documentation
general-purposeAgent polyvalent pour tâches complexes

Tavilynpm i -g tavily-mcpRecherche web intelligente avec résumés
Context7npx -y @upstash/context7-mcpDocs à jour de n'importe quelle lib/framework
Fetchuvx mcp-server-fetchRequêtes HTTP/HTTPS directes
Sequential Thinkingnpm i -g @modelcontextprotocol/server-sequential-thinkingRaisonnement en chaîne structuré

Mempalace(via marketplace)Mémoire persistante entre sessions
Config MCP (~/.claude/claude_mcp_config.json)
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    },
    "tavily": {
      "command": "npx",
      "args": ["tavily-mcp"],
      "env": { "TAVILY_API_KEY": "tvly-xxxx" }
    },
    "context7": {
      "command": "npx",
      "args": ["-y", "@upstash/context7-mcp"]
    }
  }
}

🔑 Permissions

Contrôle granulaire sur ce que Claude peut exécuter. Format : Bash(pattern*), Read(glob), Write(glob), ou nom d'outil MCP.

Allowlist Node.js — évite 90% des prompts
"allow": [
  "Bash(npm run dev)",
  "Bash(npm run build)",
  "Bash(npm run test)",
  "Bash(npm run lint)",
  "Bash(npm install *)",
  "Bash(npx serve *)",
  "Bash(node scripts/*)",
  "Bash(node --version)",
  "Bash(open *)",
  "Bash(mkdir -p *)",
  "Bash(curl -s *)",
  "Bash(ffprobe *)"
]
Deny — Sécurité absolue
Ces règles bloquent définitivement les actions, même si demandées.
"deny": [
  "Bash(rm -rf /*)",
  "Bash(git push --force *)",
  "Bash(curl * | bash)",
  "Bash(wget * | sh)",
  "Read(**/.env)",
  "Read(**/.env.*)",
  "Read(**/secrets/**)",
  "Read(**/*.pem)",
  "Read(**/*.key)",
  "Write(**/.env)",
  "Write(**/.env.*)",
  "Write(**/secrets/**)"
]
Ask — Confirmation obligatoire
"ask": [
  "Bash(git push *)",
  "Bash(rm -rf *)",
  "Bash(DROP *)",
  "Bash(DELETE FROM *)",
  "Bash(curl * | bash)"
]
Auto-allowed — Jamais de prompt
Ces commandes sont déjà auto-approuvées par Claude Code — inutile de les mettre dans allow.
ls, cat, head, tail, wcLecture de fichiers/répertoires
git status, git log, git diff, git showLecture git (tous les sous-cmds read-only)
gh pr view, gh pr list, gh run viewGitHub CLI read-only
find, grep, rg, fdRecherche fichiers
node -v, python --versionVersions outils
echo, printf, date, pwd, whoamiUtilitaires système basiques

🎨 UI & Personnalisation

Thème, spinner, raccourcis clavier. Configurer dans settings.json ou ~/.claude/keybindings.json.

Thème et verbose
// settings.json
{ "theme": "dark" }   // dark | light | auto

// Ou via commande slash
/config theme dark
/config verbose true  // affiche les détails techniques
Spinner verbs personnalisés
Messages affichés en rotation pendant que Claude travaille.
"spinnerVerbs": {
  "mode": "append",
  "verbs": [
    "🚀 Déploiement",
    "🔍 Investigation",
    "⚡ Optimisation",
    "🎨 Design en cours",
    "🧪 Tests"
  ]
}
Tips spinner personnalisés
Rappels affichés sous le spinner — utile pour tes propres règles de workflow.
"spinnerTipsOverride": {
  "excludeDefault": false,
  "tips": [
    "💡 /plan avant toute tâche 3+ étapes",
    "🎯 /model opus pour les décisions d'archi",
    "📦 /compact si contexte > 70%",
    "🔑 tasks/todo.md à jour en fin de session"
  ]
}
Raccourcis clavier (~/.claude/keybindings.json)
Créer des raccourcis pour les commandes fréquentes.
[
  { "key": "ctrl+shift+p", "command": "/compact"      },
  { "key": "ctrl+shift+o", "command": "/model opus"   },
  { "key": "ctrl+shift+s", "command": "/model sonnet" },
  { "key": "ctrl+shift+h", "command": "/model haiku"  },
  { "key": "ctrl+shift+k", "command": "/clear"        }
